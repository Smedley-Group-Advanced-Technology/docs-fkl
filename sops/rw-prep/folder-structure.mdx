## Description

How to properly create folder structures locally ahead of an event.

---

## Why?

Various tools are built to process the telemetry data and supporting files from a certain location.

---

## When?

### Pre-Event at the Factory

- Creating Folders

---

## Prerequisites

- STL installed

---

## Procedure

### Creating Folders

<Note>
  There are multiple folders that are necessary for all processes to run. These are required to be made ahead of each race weekend.
</Note>

<Note>
  This section assumes you already understand STL and the supporting folder structure.
</Note>

<Steps>
  <Step title="Navigate to the STL Database location">
    Navigate to the correct file location locally for your STL Database.

    ![Step 1](/images/sops/rw-prep/folder-structure-step-1.png)
  </Step>
  <Step title="Create a Video folder for the event">
    Under **Video**, create a folder with the correct naming convention for that event.

    ![Step 2](/images/sops/rw-prep/folder-structure-step-2.png)
  </Step>
  <Step title="Create a DataSetup folder for the event">
    Return to the top-level STL folder and navigate to **DataSetup** > **\<year\>** > **\<eventType\>** and create a folder with the correct naming convention for that event.

    ![Step 3](/images/sops/rw-prep/folder-structure-step-3.png)
  </Step>
  <Step title="Create an Automated subfolder">
    Enter that new folder and create another folder named **Automated**.

    ![Step 4](/images/sops/rw-prep/folder-structure-step-4.png)
  </Step>
</Steps>
